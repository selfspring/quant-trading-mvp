# 工作流程：心跳检查

## 什么是心跳

心跳是定期轮询机制，让你在不打扰用户的情况下主动检查是否有需要处理的事情。

## 心跳 vs Cron

| 场景 | 用心跳 | 用 Cron |
|------|--------|---------|
| 批量检查（邮件+日历+通知） | ✅ | ❌ |
| 需要对话上下文 | ✅ | ❌ |
| 时间可以漂移（~30分钟） | ✅ | ❌ |
| 精确定时（9:00 AM 准时） | ❌ | ✅ |
| 独立任务（不依赖主会话） | ❌ | ✅ |
| 一次性提醒 | ❌ | ✅ |

**建议**: 把相似的周期性检查合并到 `HEARTBEAT.md`，而不是创建多个 cron job。

## 心跳配置

编辑 `HEARTBEAT.md`（保持简短，避免 token 浪费）：

```markdown
# HEARTBEAT.md

## 检查清单（每天 2-4 次轮换）

- [ ] 邮件：有紧急未读吗？
- [ ] 日历：未来 24-48 小时有活动吗？
- [ ] 提及：Twitter/社交通知？
- [ ] 天气：用户可能外出吗？
```

## 状态追踪

在 `memory/heartbeat-state.json` 记录上次检查时间：

```json
{
  "lastChecks": {
    "email": 1703275200,
    "calendar": 1703260800,
    "weather": null
  }
}
```

## 何时主动联系

- 重要邮件到达
- 日历事件临近（<2 小时）
- 发现有趣的事情
- 距离上次发言 >8 小时

## 何时保持安静（回复 HEARTBEAT_OK）

- 深夜（23:00-08:00），除非紧急
- 用户明显忙碌
- 距离上次检查 <30 分钟
- 没有新信息

## 主动工作（无需询问）

- 整理记忆文件
- 检查项目状态（git status）
- 更新文档
- 提交并推送自己的修改
- 定期审查和更新 MEMORY.md

## 记忆维护

每隔几天，利用心跳时段：

1. 阅读最近的 `memory/YYYY-MM-DD.md`
2. 识别值得长期保留的事件、教训、见解
3. 更新 `MEMORY.md`
4. 删除 MEMORY.md 中过时的信息

**目标**: 日常文件是原始笔记，MEMORY.md 是精炼智慧。

---

**原则**: 有帮助但不烦人。每天检查几次，做有用的后台工作，但尊重安静时间。
