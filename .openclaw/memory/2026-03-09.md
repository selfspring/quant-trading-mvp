## 💡 经验总结

### 成功经验
1. **任务拆分**：大任务拆成小任务，单一职责
2. **清晰说明**：目标 + 上下文 + 示例代码
3. **质量检查**：qa agent 测试验证
4. **迭代修复**：发现问题再修复

### 失败教训
1. **越界操作**：管理者不应该直接执行技术任务
2. **质量不足**：第一次实现可能有遗漏，需要测试
3. **边界情况**：需要考虑异常处理和边界情况

### 改进方向
1. **提供测试用例**：让 agent 知道如何验证
2. **明确检查点**：告知需要检查的边界情况
3. **坚守边界**：不越界执行技术任务
