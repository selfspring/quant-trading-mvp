## 特征数不匹配 Bug 修复

### 问题
- 模型训练用 56 个特征（含 15 个 `disc_*` discovered factors），推理时只生成 47 个 → LightGBM 报错
- **根本原因**: `quant/factors/discovered_factors.py` 有系统性缩进错误（IndentationError），整个模块无法 import
- `feature_engineer.py` 中 `try/except` 吞掉了 ImportError，导致 15 个 disc_* 因子静默失败

### 缩进问题类型
1. **try 块体缺失缩进**: `try:` 后的行和 `try:` 同级（L944-949）
2. **for 循环体缺失缩进**: for body 的部分行掉到函数级（spectral_coherence_poi, oi_wavelet_energy 等）
3. **嵌套块体缺失缩进**: def/for/if 内的代码块缩进不足

### 修复方法
- 写了自动修复脚本 `scripts/fix_indent_final.py`
  - Pass 1: 修复过度缩进的函数体（8 spaces → 4 spaces）
  - Pass 2: 迭代编译修复，每次根据 SyntaxError 类型自动修正缩进
  - 7 次迭代后全部语法错误修复
- 修改 `ml_predictor.py`: 用 `model.feature_name()` 精确对齐特征列，而不是手动移除 timestamp

### 修复文件
- `quant/factors/discovered_factors.py` — 修复缩进（99 个因子全部可用）
- `quant/signal_generator/ml_predictor.py` — 特征对齐逻辑改进

### 验证
- `from quant.factors.discovered_factors import DISCOVERED_FACTORS` → 99 个因子 ✅
- 推理特征 63 个，模型需要 56 个，Missing = 0 ✅
- 端到端预测测试通过 ✅

### 教训
1. **不要吞异常**: `feature_engineer.py` 的 `except: pass` 掩盖了 import 失败，应该至少 log warning
2. **特征对齐应该用 model.feature_name()**: 而不是手动移除列，确保训练和推理完全一致
3. **自动生成的代码要验证语法**: discovered_factors.py 是 LLM 自动生成的，缩进规范需要校验
