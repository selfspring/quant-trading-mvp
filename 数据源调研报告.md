## 三、数据源详细技术方案

### 3.1 行情数据：CTP接口（上期所黄金期货AU）

#### 3.1.1 数据源介绍
- **官方名称**：综合交易平台（Comprehensive Transaction Platform）
- **开源实现**：OpenCTP（https://github.com/openctp/openctp）
- **覆盖范围**：上期所、郑商所、大商所、中金所等国内期货交易所
- **目标品种**：黄金期货（AU）

#### 3.1.2 API调用方式

**技术栈**：
- Python接口：`openctp-ctp` 或 `vnpy`
- 协议：基于TCP的私有协议

**连接流程**：
```python
# 示例代码（不实际执行）
from openctp_ctp import mdapi

class MarketDataCollector(mdapi.CThostFtdcMdSpi):
    def __init__(self):
        super().__init__()
        self.api = mdapi.CThostFtdcMdApi.CreateFtdcMdApi()
        
    def OnFrontConnected(self):
        """连接成功回调"""
        req = mdapi.CThostFtdcReqUserLoginField()
        req.BrokerID = "9999"
        req.UserID = "your_user_id"
        req.Password = "your_password"
        self.api.ReqUserLogin(req, 0)
        
    def OnRspUserLogin(self, pRspUserLogin, pRspInfo, nRequestID, bIsLast):
        """登录成功后订阅行情"""

        """接收实时行情数据"""
        print(f"合约: {pDepthMarketData.InstrumentID}")
        print(f"最新价: {pDepthMarketData.LastPrice}")
        print(f"成交量: {pDepthMarketData.Volume}")
        # 存储到TimescaleDB
```

#### 3.1.3 数据格式

**实时行情数据结构**：
```python
{
    "InstrumentID": "au2406",      # 合约代码
    "TradingDay": "20260308",      # 交易日
    "UpdateTime": "14:30:00",      # 更新时间
    "LastPrice": 520.5,            # 最新价
    "Volume": 123456,              # 成交量
    "OpenInterest": 234567,        # 持仓量
    "OpenPrice": 518.0,            # 开盘价
    "HighestPrice": 522.0,         # 最高价
    "LowestPrice": 517.5,          # 最低价
    "PreClosePrice": 519.0         # 昨收价
}
```

**K线数据（需自行聚合）**：
- 从Tick数据聚合30分钟K线
- 或使用历史数据接口获取

#### 3.1.4 频率限制
- **实时行情**：无限制，推送式
- **历史数据查询**：部分接口有频率限制（具体看券商）
- **连接数**：单账号通常限制1-2个连接

#### 3.1.5 认证信息

**需要用户注册**：✅ 是

**注册步骤**：
1. 访问 OpenCTP 官网或期货公司官网
2. 注册模拟账号（免费）
3. 获取：
   - BrokerID（券商代码）
   - UserID（用户名）
   - Password（密码）
   - 行情前置地址（如：tcp://180.168.146.187:10131）

**费用**：
- 模拟盘：完全免费
- 实盘：需开通期货账户，行情免费，交易收手续费

#### 3.1.6 技术方案总结

**优点**：
- ✅ 官方接口，数据准确可靠
- ✅ 实时推送，延迟低（毫秒级）
- ✅ 模拟盘免费，适合MVP开发

**缺点**：
- ⚠️ 需要用户注册账号
- ⚠️ 接口复杂，学习曲线陡峭
- ⚠️ 需要处理断线重连、心跳维护

**推荐方案**：
- 使用 `vnpy` 或 `openctp-ctp` 封装好的接口
- 先用模拟盘开发和测试
- 实现自动重连和异常处理机制

### 3.3 新闻数据

#### 3.3.1 新浪财经

**数据源介绍**：
- 网站：https://finance.sina.com.cn
- 类型：综合财经门户
- 特点：实时更新，内容丰富

**API调用方式**：

```python
import requests
from bs4 import BeautifulSoup
import time

def fetch_sina_finance_news():
    """
    爬取新浪财经快讯
    """
    url = "https://finance.sina.com.cn/roll/index.d.html"
    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
    }
    
    response = requests.get(url, headers=headers)
    response.encoding = 'utf-8'
    soup = BeautifulSoup(response.text, 'html.parser')
    
    news_list = []
    for item in soup.select('.list_009 li'):
        title = item.select_one('a').text.strip()
        link = item.select_one('a')['href']
        time_str = item.select_one('.time').text.strip()
        
        news_list.append({
            'source': 'sina',
            'title': title,
            'url': link,
            'time': time_str
        })
    
    return news_list
```

**数据格式**：

```python
}
```

## 五、风险评估与应对策略

### 5.1 数据源风险矩阵

| 数据源 | 风险类型 | 风险等级 | 应对策略 |
|-------|---------|---------|---------|
| CTP接口 | 需要��册 | 🟡 中 | 提供注册指引，使用模拟盘 |
| TuShare | 积分限制 | 🟡 中 | 使用AKShare作为主力 |
| AKShare | 源网站变化 | 🟡 中 | 准备多个备用源 |
| 新浪财经 | 反爬虫 | 🟡 中 | 控制请求频率，轮换UA |
| 东方财富 | 反爬虫 | 🟢 低 | 使用AKShare封装 |
| 金十数据 | 反爬虫强 | 🔴 高 | 谨慎使用，准备IP代理 |
| 路透RSS | 不可用 | 🔴 高 | 放弃使用 |
| 彭博RSS | 付费 | 🔴 高 | 放弃使用 |
| Twitter API | 限制严格 | 🔴 高 | 放弃使用 |

### 5.2 应对策略

#### 策略1：多源备份
- 基本面数据：AKShare（主） + TuShare（备）
- 新闻数据：东方财富（主） + 新浪财经（备） + 金十数据（可选）

#### 策略2：反爬虫对策
- User-Agent轮换
- 请求频率控制（3-5秒间隔）
- IP代理池（如果需要）
- 使用官方API优先（如AKShare）

#### 策略3：数据验证
- 关键数据双源验证
- 异常值检测
- 数据完整性检查

#### 策略4：降级方案
- 当实时数据源失效时，使用历史数据
- 当新闻源失效时，降低新闻信号权重
- 保持系统可用性优先

## 六、总结与建议

### 6.1 可直接使用的数据源

✅ **推荐使用**：
1. **AKShare**：基本面数据首选，无需注册，完全免费
2. **东方财富**（通过AKShare）：新闻数据首选，稳定可靠
3. **新浪财经**：新闻数据备选，直接爬取

### 6.2 需要用户注册的数据源

⚠️ **需要用户操作**：
1. **CTP接口（OpenCTP）**：
   - 用途：黄金期货实时行情
   - 注册难度：中等
   - 建议：使用模拟盘，提供详细注册指引
   
2. **TuShare**：
   - 用途：基本面数据备用
   - 注册难度：简单
   - 建议：可选，AKShare已足够

### 6.3 不推荐使用的数据源

❌ **放弃使用**：
1. **路透RSS**：访问失败
2. **彭博RSS**：需要付费
3. **Twitter API**：免费版限制过严

### 6.4 开发优先级

**Phase 1（核心功能）**：
1. ✅ AKShare基本面数据采集
2. ✅ 东方财富新闻采集（通过AKShare）
3. ⚠️ CTP行情数据采集（需用户注册）

**Phase 2（增强功能）**：
1. 新浪财经新闻爬取
2. TuShare备用数据源
3. 数据验证和去重机制

**Phase 3（可选功能）**：
1. 金十数据快讯（谨慎使用）
2. IP代理池
3. 更多数据源扩展

1. **CTP接口是唯一需要用户注册的核心数据源**
