# 量化交易系统 MVP - 项目总结文档

## 一、项目背景

### 1.1 项目定位
基于AI收集分析的多元市场量化交易系统，采用MVP（最小可行产品）策略进行开发。

### 1.2 市场选择
- **初期目标**：上期所黄金期货（AU）
- **扩展计划**：后续扩展至其他市场品种

### 1.3 核心价值主张
- 多信号融合决策（技术指标 + 机器学习 + AI新闻解读）
- 完整风控体系保障资金安全
- 本地部署，数据安全可控
- Web界面实时监控

---

## 二、核心需求

### 2.1 交易策略需求
- **交易周期**：30分钟K线波段交易
- **反应机制**：重大新闻事件快速响应
- **决策方式**：多信号融合（技术指标 + ML模型 + LLM新闻解读）

### 2.2 风控需求
- 动态仓位管理（根据信号强度调整）
- 止损止盈机制（由ML模型给出）
- 波动率过滤（ATR指标）
- 连败熔断机制
- 最大回撤控制（单周不超过25%）

### 2.3 数据需求
- **行情数据**：CTP实时行情接口（30分钟K线）
- **基本面数据**：美联储利率、非农数据、CPI、美元指数、美债收益率
- **新闻数据**：财经快讯、RSS订阅、社交媒体（Twitter/X）
- **约束条件**：所有数据源必须免费

### 2.4 监控需求
- Web界面实时展示持仓、盈亏、信号状态
- 邮件告警（交易触发、连续亏损、系统异常）
- 历史交易记录查询
- 回测结果可视化

### 3.3 核心算法设计

#### 3.3.1 技术指标模块
- **工具库**：TA-Lib
- **指标集**：MA、MACD、RSI、布林带、ATR
- **输入**：30分钟K线数据
- **输出**：实时计算的指标值

#### 3.3.2 机器学习模块（LightGBM）
- **任务类型**：回归（预测未来1小时收益率）
- **特征工程**：
  - 价格特征：开高低收、成交量、持仓量
  - 技术指标：MA、MACD、RSI等
  - 基本面特征：美元指数、美债收益率等
- **标签定义**：`(future_close - current_close) / current_close`
- **训练数据**：2020-2023年训练，2024年验证
- **输出**：预测收益率 + 置信度

#### 3.3.3 LLM新闻解读模块（Claude Opus 4）
- **输入**：实时抓取的新闻原文
- **输出**：结构化JSON
  ```json
  {
    "importance": "high|medium|low|irrelevant",
    "direction": "bullish|bearish|neutral",
    "timeframe": "immediate|short-term|long-term",
    "confidence": 0.0-1.0,
    "reasoning": "简短解释"
  }
  ```
- **触发规则**：
  - `importance >= medium` 且 `confidence >= 0.6`：记录
  - `importance == high` 且 `confidence >= 0.75`：立即触发交易信号

#### 3.3.4 多信号融合算法
- **加权综合**：
  - ML模型：50%
  - 技术指标：30%
  - LLM新闻：20%
- **一致性检查**：至少2/3子信号方向一致才输出交易信号
- **输出**：交易方向（买入/卖出/观望）+ 信号强度（0.0-1.0）

### 3.4 风控算法设计

#### 3.4.1 信号过滤
- **波动率过滤**：`ATR > 近20日均值 * 2` 时降低仓位50%或暂停
- **信号强度阈值**：ML置信度 < 0.65 时不交易
- **连败熔断**：连续3笔亏损后暂停1小时

#### 3.4.2 动态仓位管理
| 信号强度 | 仓位比例 |
|---------|---------|
| 0.65-0.75 | 30% |
| 0.75-0.85 | 50% |
| > 0.85 | 70% |
| 最大仓位 | 70% |

#### 3.4.3 止损止盈
- 由ML模型预测收益率基础上设定
- 盈亏比：2:1
- 示例：预测+0.8%，止盈+1.0%，止损-0.4%

#### 3.4.4 最大回撤控制
- 单周最大亏损：≤ 25%
- 触发后：暂停交易 + 邮件告警

## 四、关键功能点

### 4.1 数据采集层

#### 功能1：行情数据采集
- **数据源**：CTP实时行情接口
- **采集频率**：实时（30分钟K线）
- **数据内容**：开高低收、成交量、持仓量
- **存储方案**：TimescaleDB

#### 功能2：基本面数据采集
- **数据源**：TuShare、AKShare等免费数据源
- **采集频率**：定时爬取（每日/每周）
- **数据内容**：美联储利率、非农、CPI、美元指数、美债收益率
- **存储方案**：TimescaleDB

#### 功能3：新闻数据采集
- **数据源**：
  - 新浪财经、东方财富、金十数据（实时快讯）
  - 路透社、彭博RSS
  - Twitter/X（财经大V、美联储官员）
- **采集频率**：实时监控
- **存储方案**：
  - 原文：TimescaleDB
  - Embedding：Chroma（用于相似度检索）
- **重要约束**：
  - AI负责发现数据源、生成采集脚本
  - AI不直接爬取数据
  - 需要注册的由用户操作

### 4.2 信号生成层

#### 功能4：技术指标计算
- 实时计算MA、MACD、RSI、布林带、ATR
- 基于30分钟K线数据
- 输出标准化的指标值

#### 功能5：ML模型预测
- 加载训练好的LightGBM模型
- 实时特征工程（价格+技术指标+基本面）
- 输出预测收益率和置信度

#### 功能6：LLM新闻解读
- 接收实时新闻数据
- 调用Claude Opus 4 API进行解读
- 输出结构化的重要性、方向、时间框架、置信度
- **关键约束**：不使用启发式规则平替AI判断

#### 功能7：多信号融合
- 收集技术指标、ML预测、LLM解读结果
- 加权综合（可配置权重）
- 方向一致性检查
- 输出最终交易信号和强度

### 4.3 风控层

#### 功能8：信号过滤
- 波动率检查（ATR）
- 信号强度阈值过滤
- 连败熔断机制

#### 功能9：仓位管理
- 根据信号强度动态调整仓位
- 最大仓位限制（70%）

#### 功能10：止损止盈
- 基于ML预测设定止损止盈点位
- 实时监控价格触发

#### 功能11：回撤控制
- 实时计算单周亏损
- 触发阈值后暂停交易并告警

### 4.4 交易执行层

#### 功能12：CTP接口封装
- 实时行情订阅
- 下单（开仓/平仓）
- 订单状态跟踪
- 成交回报处理
- MVP阶段使用模拟盘

#### 功能13：订单管理
- 限价单下单（避免滑点）
- 订单状态管理（待提交、已提交、成交、撤单、失败）
- 异常处理（重试、超时撤单）

### 4.5 回测框架

#### 功能14：Backtrader集成
- 加载历史数据（K线、基本面、新闻）
- 回测周期：2020-2024年
- 评估指标：年化收益率、夏普比率、最大回撤、胜率、盈亏比
- 样本外测试：2020-2023训练，2024验证

### 4.6 监控与告警

#### 功能15：Web监控界面
- **实时数据**：
  - 当前持仓
  - 实时盈亏
  - 最新信号状态
  - 账户资金曲线
- **历史数据**：
  - 交易记录（时间、价格、盈亏、决策依据）
  - 回测结果展示
- **技术实现**：WebSocket实时推送

#### 功能16：邮件告警
- **触发条件**：
  - 触发交易（开仓/平仓）
  - 连续亏损3次
  - 单周亏损超过20%
  - 系统异常（数据源失效、模型推理失败）
- **告警内容**：事件类型、详细信息、时间戳

## 六、关键约束与交付标准

### 6.1 关键约束

1. **AI不降级**：在需求文档标注"AI"的位置，不使用启发式规则平替
2. **免费数据源**：所有数据源必须免费，需要注册的由用户操作
3. **本地部署**：系统运行在本地，不依赖云服务（除了LLM API）
4. **模拟盘先行**：真实交易前必须通过模拟盘验证

### 6.2 交付标准

- ✅ 代码可运行，有完整的README
