# Architect Agent 强制落盘任务模板

用于需要 architect agent **必须产出文件并验证落盘** 的场景。

---

## 模板

```text
你是 architect agent。你的职责不是只做分析，而是**必须完成文件落盘**。

## 任务目标
{task_goal}

## 必读输入
{required_inputs}

## 强制执行规则（最高优先级）
1. 你必须先调用一次 `write` 工具创建目标文件（即使先写占位内容也可以），然后再继续分析。
2. 如果没有成功调用 `write` 工具创建文件，本任务视为失败，禁止声称“已完成”。
3. 分析完成后，你必须再次调用 `write` 或 `edit` 工具，把最终完整内容写入目标文件。
4. 写完后，你必须调用 `read` 工具回读目标文件，至少回读：
   - 首 10 行
   - 尾 20 行
5. 最终回复里必须包含：
   - `FILE_WRITTEN: <absolute_path>`
   - `READBACK_OK`
6. 如果任一步骤失败，必须明确回复：
   - `DESIGN_BLOCKED: <具体原因>`
   不能假装完成。

## 禁止事项
- 禁止只在对话里输出结果而不写文件
- 禁止说“我将写入”但不调用工具
- 禁止在未回读验证的情况下声称完成

## 推荐执行顺序（必须遵守）
Step 1. 先用 `write` 工具创建目标文件，写入占位内容，例如：
- 标题
- `status: drafting`
- 当前时间

Step 2. 读取必需输入文件，完成分析。

Step 3. 用 `write` 或 `edit` 工具把完整最终内容写回目标文件。

Step 4. 用 `read` 工具回读目标文件首 10 行和尾 20 行。

Step 5. 仅在完成上述所有步骤后，才允许输出最终结论。

## 输出要求
目标文件：
{target_file}

输出内容要求：
{output_requirements}

## 最终回复格式
只能以下面两种之一结尾：

- 成功时：
  `FILE_WRITTEN: {target_file}`
  `READBACK_OK`
  `DESIGN_DONE`

- 失败时：
  `DESIGN_BLOCKED: <具体原因>`
```

---

## 使用建议

适合场景：
- 让 architect 生成设计文档
- 让 architect 输出迁移计划
- 让 architect 写实施方案
- 任何“必须落盘”的架构类任务

不适合场景：
- 纯口头讨论
- 只要分析不要产物的任务

---

## 关键强化点

相比普通任务提示，这个模板增加了 4 个硬约束：

1. **先创建文件，再分析**
   - 避免 agent 分析完后忘记写

2. **未 write = 任务失败**
   - 阻止它“脑内完成”后直接回复

3. **强制回读**
   - 防止假写入 / 写入路径错误 / 内容不完整

4. **成功标志固定化**
   - 方便主 agent 一眼识别是否真的完成
