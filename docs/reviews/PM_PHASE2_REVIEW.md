## 三、缺失与妥协

**为跑通主链路做的妥协**:
- 🔴 **实际 CTP 发单**: `trade_executor` 只生成 Order 对象，未调用 `tdapi.ReqOrderInsert`
- 🔴 **多信号融合**: PRD 要求技术指标+ML+LLM 动态权重融合，目前只有 ML 单信号驱动
- 🔴 **市场状态识别**: ADX+ATR 判断趋势/震荡/高波动，未实现
- 🔴 **多进程编排**: 无 Supervisor 配置，无进程启动脚本，各模块独立但未串联

**完全缺失的功能**:
- ❌ LLM 情绪分析（Claude 新闻解读）— 未开始
- ❌ 新闻爬虫 + 基本面数据采集 — 未开始
- ❌ Web 监控界面（Vue 3 + FastAPI）— 未开始
- ❌ 邮件告警 — 未开始
- ❌ VectorBT 回测框架 — 未开始
- ❌ `technical_signals.py` 只有文件头，无实现

## 四、下一步建议（MVP 快速验证优先）

**建议优先级**: 先跑通盘中策略，再补监控。

1. **P0 - 补齐发单闭环**（1-2h）: `trade_executor` 对接 `tdapi`，模拟盘验证下单
2. **P0 - 多进程串联**（2-3h）: 写一个主启动脚本，把行情→信号→风控→发单串成可运行的 pipeline
3. **P1 - 多信号融合**（2h）: 实现 `technical_signals.py`，加上动态权重融合逻辑
4. **P2 - Web 监控**: 等主链路跑通并在模拟盘验证后再做

> 核心判断：目前离"模拟盘能跑起来"只差发单对接和进程串联，建议集中火力在这两件事上。
