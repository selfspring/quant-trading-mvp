# 2026-03-13 - SimNow Tick 数据问题解决

## 问题
SimNow 仿真环境不推送实时 Tick 数据，导致无法合成 K 线，策略无法运行。

### 已验证的失败
- 测试 3 组看穿式前置 (30011/30012/30013) → 连接✅ 登录✅ 订阅✅ 但 **Tick 回调不触发**
- vnpy_ctp 握手失败 (error 4040)
- 快期客户端有行情 → 说明使用了非 CTP 数据源

## 解决方案
实现三级数据回退架构：

1. **Level 1**: SimNow 实时 Tick → 聚合 K 线（优先）
2. **Level 2**: Tushare/AkShare API → 历史 K 线（自动回退）
3. **Level 3**: 模拟数据生成 → 策略逻辑验证（保证可测试）

## 完成的工作

### 1. 创建备用数据源模块
- 文件：`quant/data_collector/backup_data_source.py`
- 支持：AkShare（默认）、Tushare（可选）、CSV、模拟生成
- 核心方法：
  - `get_realtime_klines()` - 获取 K 线
  - `generate_mock_klines()` - 生成模拟数据
  - `fetch_and_store()` - 获取并存储到数据库

### 2. 创建增强版主策略
- 文件：`scripts/main_strategy_enhanced.py`
- 功能：
  - 启动时自动检测 SimNow 数据状态
  - 无数据时自动切换备用模式
  - 策略循环支持多级回退

### 3. 测试验证
- 备用数据源工作正常（AkShare 成功获取 au2606 数据）
- 数据库已有 20 条测试 K 线
- 模拟数据生成正常

## 使用方法

```bash
# 运行增强版策略
python scripts/main_strategy_enhanced.py

# 测试备用数据源
python scripts/test_backup_data.py
```
