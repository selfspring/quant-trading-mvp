## Bug 修复：信号融合层 int/string 不兼容（第二轮）
- **commit dfa2627**: Fix signal fusion int/string normalization
- **根因**: `run_single_cycle.py` 构造 `technical_signal` 时直接传入 `ml_signal.get('signal', 'hold')`（整数），但 `signal_fusion.py::_normalize_signals()` 只认字符串 `buy/sell/hold`，导致有效 technical signal 被丢弃，融合结果更容易落到 hold。
- **修复**:

  2. `signal_fusion.py`: `_normalize_signals()` 兼容 int(1/-1/0) 和 str(buy/sell/hold)
  3. `signal_fusion.py`: `save_to_db()` 异常日志改为真实打印异常内容
  4. ruff I001 import 排序一并修复
