# 信号生成层输出分析报告

## 1. ML 预测器输出

根据日志 `logs/main_strategy.log` 的记录，ML 预测器的输出格式如下：

```python
{
    'prediction': 0.011910455918341528,  # 预测的对数收益率
    'confidence': 0.5955227959170764,    # 置信度 (0-1)
    'signal': 1                          # 信号方向 (1=做多, -1=做空)
}
```

### 关键字段说明

- **prediction**: 模型预测的未来 60 分钟对数收益率
  - 正值 → 预期上涨 → 做多信号
  - 负值 → 预期下跌 → 做空信号
  - 当前值: `0.0119` (约 1.19% 的预期收益)

- **confidence**: 置信度，基于预测值的绝对值计算
  - 公式: `confidence = min(abs(prediction) * 50, 1.0)`
  - 当前值: `0.5955` (59.55%)
  - 阈值: `0.65` (65%)

- **signal**: 信号方向
  - `1` = 做多 (buy)
  - `-1` = 做空 (sell)

## 2. 信号处理器输出

信号处理器 (`SignalProcessor`) 对 ML 预测进行过滤：

### 当前状态
```
置信度检查: 0.5955 < 0.65 ❌
结果: 信号被过滤，不生成交易意图
```

### 日志输出
```
2026-03-12 15:09:58,728 [INFO] 3. 信号处理...
2026-03-12 15:09:58,728 [INFO] 置信度不足 (0.60 < 0.65)，忽略信号
```

### 如果置信度通过，输出格式为：
```python
{
    'direction': 'buy',           # 交易方向 (buy/sell)
    'quantity': 1,                # 交易数量
    'symbol': 'au2606',           # 合约代码
    'confidence': 0.5955,         # 置信度
    'prediction': 0.0119          # 原始预测值
}
```

## 3. 问题分析

### 当前问题
**置信度不足导致所有信号被过滤**

- 当前置信度: `0.5955` (59.55%)
- 阈值要求: `0.65` (65%)
- 差距: `0.0545` (5.45%)

### 置信度计算
```
confidence = min(abs(prediction) * 50, 1.0)
confidence = min(abs(0.0119) * 50, 1.0)
confidence = min(0.5955, 1.0)
confidence = 0.5955
```

### 要达到 0.65 阈值，需要：
```
0.65 = abs(prediction) * 50
prediction = 0.65 / 50
prediction = 0.013
```

**当前预测值 (0.0119) 略低于所需值 (0.013)**

## 4. 解决方案

### 方案 1: 降低置信度阈值 (推荐)
```python
# config.yaml
strategy:
  confidence_threshold: 0.55  # 从 0.65 降低到 0.55
```

**优点**: 立即生效，可以让当前模型的信号通过
**缺点**: 可能增加假信号

### 方案 2: 改进模型训练
- 增加更多特征
- 调整模型参数
- 使用更长的训练数据
- 尝试其他模型 (XGBoost, CatBoost)

**优点**: 从根本上提升预测质量
**缺点**: 需要重新训练，耗时较长

### 方案 3: 调整置信度计算公式
```python
# 当前公式
confidence = min(abs(prediction) * 50, 1.0)

# 可选方案
confidence = min(abs(prediction) * 40, 1.0)  # 降低乘数
confidence = min(abs(prediction) * 60, 1.0)  # 提高乘数
```

**优点**: 灵活调整置信度映射
**缺点**: 需要修改代码
