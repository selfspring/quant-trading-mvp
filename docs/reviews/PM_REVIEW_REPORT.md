### 3. 不符合项

#### 3.1 技术栈不一致

**不符合描述**: CTP 接口使用 openctp-ctp，而非 PRD 要求的 vnpy-ctp

- **PRD 要求**: vnpy-ctp 6.6.9+（PRD-v2.md 明确指定）
- **实际情况**: 使用 openctp-ctp（`quant/data_collector/ctp_market.py`）
- **差异原因**: 可能是开发时选择了更轻量的库
- **建议**: 
  - **选项A（推荐）**: 更新 PRD，接受 openctp-ctp（如果功能满足需求）
  - **选项B**: 重构为 vnpy-ctp（更成熟，社区支持更好）

**影响**: 中等。openctp-ctp 功能可能不如 vnpy-ctp 完善，但对 MVP 阶段影响有限。

#### 3.2 配置文件格式不一致

**不符合描述**: 配置文件使用 .env + pydantic，而非 YAML

- **PRD 要求**: 未明确指定，但提供了 `config.example.yaml`
- **实际情况**: 使用 `.env` + `pydantic-settings`（`quant/common/config.py`）
- **差异原因**: pydantic-settings 提供更好的类型安全和验证
- **建议**: 更新 PRD，明确推荐 pydantic-settings 方案

**影响**: 低。实际方案更优，但需要更新文档。

#### 3.3 进程管理方案未实现

**不符合描述**: 缺少 Supervisor 配置和多进程启动脚本

- **PRD 要求**: 使用 Supervisor 管理 5 个进程（PRD-v3.md#系统架构）
- **实际情况**: 未找到 Supervisor 配置文件和启动脚本
- **差异原因**: Milestone 1 尚未完成
- **建议**: 立即实现 `start_local.py` 和 `supervisord.conf`

**影响**: 高。阻塞多进程架构验证。
