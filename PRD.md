# 量化交易系统 MVP - 产品需求文档

## 项目概述

基于AI收集分析的多元市场量化交易系统，MVP阶段聚焦上期所黄金期货（AU），后续扩展至其他市场。

## 核心目标

- 30分钟K线波段交易 + 重大新闻事件快速反应
- 多信号融合决策（技术指标 + ML模型 + LLM新闻解读）
- 完整风控体系
- 本地部署，Web监控界面

## 技术栈

- **语言**: Python 3.10+
- **数据库**: TimescaleDB（时序数据）+ Chroma（向量数据库）
- **ML框架**: LightGBM
- **回测框架**: Backtrader
- **交易接口**: CTP（上期所）
- **LLM**: Claude Opus 4
- **前端**: 待Architect确定（建议React/Vue + WebSocket实时推送）

## 功能模块

### 1. 数据采集层

#### 1.1 行情数据
- **来源**: CTP实时行情接口
- **频率**: 30分钟K线（实时）
- **内容**: 开高低收、成交量、持仓量
- **存储**: TimescaleDB

#### 1.2 基本面数据
- **来源**: 免费数据源（TuShare、AKShare等）
- **内容**: 
  - 美联储利率决议
  - 非农就业数据
  - CPI通胀数据
  - 美元指数
  - 美债收益率
- **频率**: 定时爬取（每日/每周）
- **存储**: TimescaleDB

#### 1.3 新闻数据
- **来源**: 
  - 新浪财经、东方财富、金十数据（实时快讯）
  - 路透社、彭博RSS
  - Twitter/X（财经大V、美联储官员）
- **频率**: 实时监控
- **存储**: 
  - 原文: TimescaleDB
  - Embedding: Chroma（用于相似度检索）

**重要约束**: 
- 所有数据源必须免费
- AI负责发现数据源、生成采集脚本，但不直接爬取
- 需要注册的由用户操作

### 2. 信号生成层

#### 2.1 技术指标
- **库**: TA-Lib
- **指标**: 
  - 均线（MA）
  - MACD
  - RSI
  - 布林带（Bollinger Bands）
  - ATR（平均真实波幅，用于波动率过滤）
- **输入**: 30分钟K线
- **输出**: 实时计算的指标值

#### 2.2 ML模型（LightGBM）
- **任务类型**: 回归（预测未来2根30分钟K线的收益率）
- **特征工程**: 
  - 价格数据（开高低收、成交量、持仓量）
  - 技术指标（MA、MACD、RSI等）
  - 基本面数据（美元指数、美债收益率等）
- **标签定义**: 未来2根K线（1小时后）的收益率
  - 公式: `(future_close - current_close) / current_close`
- **训练数据**: 历史数据（2020-2023年训练，2024年验证）
- **输出**: 
  - 预测收益率（连续值，如+0.8%或-0.5%）
  - 置信度（模型内部评估）

#### 2.3 LLM新闻解读（Claude Opus 4）
- **输入**: 实时抓取的新闻原文
- **输出**: 结构化JSON
  ```json
  {
    "importance": "high|medium|low|irrelevant",
    "direction": "bullish|bearish|neutral",
    "timeframe": "immediate|short-term|long-term",
    "confidence": 0.0-1.0,
    "reasoning": "简短解释"
  }
  ```
- **触发条件**: 
  - `importance >= medium` 且 `confidence >= 0.6` 时记录并可能触发交易
  - `importance == high` 且 `confidence >= 0.75` 时立即触发交易信号

**重要约束**: 
- 不使用启发式规则平替AI判断
- 全部新闻都送给LLM判断，不做关键词预过滤

#### 2.4 多信号融合
- **输入**: 
  - 技术指标信号
  - ML模型预测值+置信度
  - LLM新闻解读结果
- **融合逻辑**: 
  - 加权综合（权重可配置，初始建议: ML 50%, 技术指标 30%, LLM 20%）
  - 方向一致性检查: 至少2/3的子信号方向一致才输出交易信号
- **输出**: 
  - 交易方向（买入/卖出/观望）
  - 信号强度（0.0-1.0，用于动态调仓）

### 3. 风控层

#### 3.1 信号过滤
- **波动率过滤**: 
  - 计算ATR，如果 `ATR > 近20日均值 * 2`，降低仓位50%或暂停交易
- **信号强度阈值**: 
  - ML模型置信度 < 0.65 时不交易
- **连败熔断**: 
  - 连续3笔交易亏损后，暂停交易1小时

#### 3.2 仓位管理
- **动态调仓**: 根据信号强度调整仓位
  - 信号强度 0.65-0.75: 30%仓位
  - 信号强度 0.75-0.85: 50%仓位
  - 信号强度 > 0.85: 70%仓位
- **最大仓位**: 不超过总资金的70%

#### 3.3 止损止盈
- **由ML模型给出**: 
  - 模型预测收益率的基础上，设定止损/止盈点位
  - 例如: 预测+0.8%，止盈设在+1.0%，止损设在-0.4%（盈亏比2:1）

#### 3.4 最大回撤控制
- **单周最大亏损**: 不超过总资金的25%
- **触发后**: 暂停交易，发送邮件告警

### 4. 交易执行层

#### 4.1 CTP接口
- **功能**: 
  - 实时行情订阅（30分钟K线）
  - 下单（开仓/平仓）
  - 订单状态跟踪
  - 成交回报处理
- **模式**: 
  - MVP阶段使用模拟盘
  - 验证通过后切换真实盘

#### 4.2 订单管理
- **订单类型**: 限价单（避免滑点过大）
- **订单状态**: 待提交、已提交、部分成交、全部成交、已撤单、失败
- **异常处理**: 
  - 下单失败重试（最多3次）
  - 超时未成交自动撤单

### 5. 回测框架

#### 5.1 Backtrader集成
- **数据源**: 
  - 历史K线数据（TuShare/AKShare）
  - 历史基本面数据
  - 历史新闻数据（如果有）
- **回测周期**: 2020-2024年
- **评估指标**: 
  - 年化收益率
  - 夏普比率（收益/风险比）
  - 最大回撤
  - 胜率
  - 盈亏比（平均盈利/平均亏损）
- **样本外测试**: 
  - 2020-2023年训练
  - 2024年测试（验证是否过拟合）

### 6. 监控与告警

#### 6.1 Web监控界面
- **实时数据**: 
  - 当前持仓
  - 实时盈亏
  - 最新信号状态（技术指标、ML预测、LLM解读）
  - 账户资金曲线
- **历史数据**: 
  - 交易记录（每笔交易的开平仓时间、价格、盈亏、决策依据）
  - 回测结果展示
- **技术**: WebSocket实时推送

#### 6.2 邮件告警
- **触发条件**: 
  - 触发交易（开仓/平仓）
  - 连续亏损3次
  - 单周亏损超过20%
  - 系统异常（数据源失效、模型推理失败等）
- **内容**: 
  - 事件类型
  - 详细信息（价格、仓位、原因等）
  - 时间戳

## 关键约束

1. **AI不降级**: 在需求文档标注"AI"的位置，不使用启发式规则平替
2. **免费数据源**: 所有数据源必须免费，需要注册的由用户操作
3. **本地部署**: 系统运行在本地，不依赖云服务（除了LLM API）
4. **模拟盘先行**: 真实交易前必须通过模拟盘验证

## 交付标准

- 代码可运行，有完整的README
- 关键模块有单元测试
- 回测结果达到预期（具体指标待定）
- Web界面可访问，数据实时更新
- 文档完整（架构文档、API文档、部署文档）

## 时间预期
