# 2026-03-12 - 量化交易系统调试日志

## 核心发现：SimNow CTP 行情推送机制

### 问题
- SimNow CTP 前置服务器连接✅ 登录✅ 订阅✅ 但收不到 Tick 数据❌
- 测试了 3 组看穿式前置 (30011/30012/30013)，结果相同
- vnpy_ctp 握手失败 (error 4040 解密错误)

### 根本原因
**SimNow 只在交易时段推送 Tick 数据**：
- 早盘：09:00-11:30
- 午盘：13:30-15:00  
- 夜盘：21:00-02:30

白天非交易时段 (12:00-13:30, 15:00-21:00) 不推送任何数据，这是正常行为。

### 解决方案
1. 使用历史数据测试策略逻辑 ✅
2. 夜盘时段 (21:00 后) 测试实时行情推送
3. 备选：接入 Tushare/AkShare 获取实时数据

## 技术细节

### 数据库 interval 字段问题
- **问题**：查询返回 0 行，但数据库有 200 条记录
- **原因**：代码中 `interval_map` 把 '1min' 映射成 '1m'，但数据库存的是 '1min'
- **解决**：`UPDATE kline_data SET interval = '1m' WHERE interval = '1min'`

### CTP API 版本

- 官方 API: E:\v6.7.11_P4_20251120_traderapi (C++ 版本)

## 经验教训
1. SimNow 仿真环境的行情推送**严格遵循交易时段**
2. 快期客户端能看到行情 ≠ CTP API 能收到 Tick (数据源不同)
3. 数据库字段映射要保持一致 (interval: '1min' vs '1m')
4. Python CTP 包版本兼容性很重要 (vnpy_ctp 不兼容 SimNow)
