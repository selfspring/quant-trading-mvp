## 11. Delivery 交付文件
implementation / 设计类任务完成后，必须生成一个可引用的 delivery 文件。

建议路径：
- `docs/refactor-status/deliveries/YYYY-MM-DD-HH-<task-name>.md`

文件至少包含：
- 任务名称
- 任务范围
- 结果摘要
- 修改文件
- 与计划/目标对齐
- 验证过程
- 风险与遗留
- 下一步建议

如果当前任务需要进入 QA，则 delivery 文件路径必须先写入状态文件或明确提供给主控 agent。

## 12. 强制验证清单
任务结束前必须逐条完成：

- [ ] 读回所有修改过的关键文件
- [ ] 对照任务目标逐条检查
- [ ] 说明哪些目标已完成，哪些未完成
- [ ] 运行相关测试 / 检查 / dry-run（如适用）
- [ ] 汇报实际结果，不得只说“应该可以”
- [ ] 标出风险、假设、遗留问题
- [ ] 给出建议下一步

禁止：
- 不读回文件就宣告完成
- 不做检查就说“没问题”
- 把猜测当结果汇报
