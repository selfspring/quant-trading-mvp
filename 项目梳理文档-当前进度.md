## 一、项目概览

### 1.1 项目目标

构建一个基于AI的多元市场量化交易系统，聚焦上期所黄金期货（AU）30分钟K线波段交易。

**核心价值**：
- 多信号融合决策（技术指标 30% + ML模型 50% + LLM新闻解读 20%）
- 动态权重调整（根据市场状态自适应）
- 完整风控体系 + 故障降级方案
- 全链路消息追踪（可观测性）

### 1.2 技术栈

| 类别 | 技术选型 | 用途 |
|-----|---------|------|
| 语言 | Python 3.10+ | 主开发语言 |
| 时序数据库 | TimescaleDB 2.x | 存储K线、指标、交易数据 |
| 向量数据库 | Chroma 0.4+ | 新闻embedding，LLM缓存 |
| 消息队列 | Redis 7.x | 进程间通信 |
| 交易接口 | vnpy-ctp 6.6.9+ | 连接上期所CTP |
| 机器学习 | LightGBM 4.x | 价格预测 |
| 回测框架 | VectorBT 0.26+ | 策略回测（快100倍） |
| AI模型 | Claude Opus 4 | 新闻解读 |
| Web框架 | FastAPI + Vue 3 | 监控界面 |
| 进程管理 | Supervisor | 进程守护 |

## 四、已完成的基础设施

### 4.1 配置管理（config.py）

**功能**：
- 使用 pydantic-settings 实现类型安全的配置
- 支持环境变量覆盖
- 包含所有子系统配置（数据库、Redis、CTP、Claude等）

**使用示例**：
```python
from quant.common.config import config

# 访问配置
db_host = config.database.host
ctp_account = config.ctp.account_id
```

**环境变量配置**：
```bash
# .env 文件
DATABASE__PASSWORD=your_password
CTP__ACCOUNT_ID=your_account
CLAUDE__API_KEY=your_api_key
```

### 4.2 消息追踪（tracer.py）

**功能**：
- 全链路消息追踪（trace_id）
- 自动记录事件到数据库和Redis
- 性能分析（latency_ms）
- 支持父子关联（parent_trace_id）

**使用示例**：
```python
from quant.common.tracer import MessageTracer

tracer = MessageTracer(redis_client, db_client, 'data_collector')

# 发布消息时自动注入 trace_id
trace_id = tracer.publish_with_trace(
    channel='kline_update',
    message={'symbol': 'au2406', 'price': 520.5}
)

# 记录事件
tracer.trace_event(
    trace_id=trace_id,
    event_type='kline_received',
    event_data={'symbol': 'au2406'},
    status='success'
)
```

### 4.3 数据库设计（14张表）

**核心表**：
1. `kline_data` - K线数据（TimescaleDB hypertable）
2. `news_raw` - 新闻原文（含去重索引 content_hash）
3. `news_analysis` - AI新闻解读结果
4. `technical_indicators` - 技术指标（含扩展字段 JSONB）
5. `ml_predictions` - ML模型预测
6. `trading_signals` - 交易信号
7. `orders` - 订单记录
8. `trades` - 成交记录
9. `positions` - 持仓记录
10. `account_snapshot` - 账户快照
11. `data_quality_log` - 数据质量监控 ⭐
12. `signal_performance` - 信号回溯分析 ⭐
13. `message_trace` - 消息流追踪 ⭐
14. `fundamental_data` - 基本面数据

**初始化命令**：
```bash
python scripts/init_db.py
```

### 4.4 依赖管理（requirements.txt）

**核心依赖**：
- psycopg2-binary（PostgreSQL）
- redis（消息队列）
- chromadb（向量数据库）
- vnpy + vnpy-ctp（交易接口）
- lightgbm（机器学习）
- vectorbt（回测）
- anthropic（Claude API）
- fastapi + uvicorn（Web框架）

**安装命令**：
```bash
pip install -r requirements.txt
```

## 六、风险提示

### 6.1 当前阶段风险

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| **CTP 账号注册失败** | 无法采集行情 | 提供详细注册指南 |
| **新闻爬虫被封IP** | 数据缺失 | 准备多个数据源 |
| **依赖安装失败** | 无法运行 | 提供 Docker 方案 |
| **数据库连接失败** | 无法存储数据 | 检查配置和权限 |

### 6.2 需要注意的事项

1. **模拟盘先行**：不要急于上真实盘，至少模拟盘运行1个月
2. **成本控制**：LLM API 建议设置每日调用上限（500次）
3. **数据备份**：定期备份数据库
4. **日志监控**：关注 logs/ 目录下的日志文件
