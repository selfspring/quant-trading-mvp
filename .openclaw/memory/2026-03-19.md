# 2026-03-19 记录

## openclaw.json 乱码修复
- **根本原因**: 用 PowerShell `Set-Content` 写含中文的 JSON 文件，导致 UTF-8 被 GBK 解码，agent id/name 出现乱码（如 `浜ゆ槗` = `交易`，`寮€鍙?` = `开发`）
- **修复方法**: 用 write 工具直接重写 openclaw.json，避免 PowerShell 编码问题
- **教训**: 永远不要用 PowerShell `Set-Content` 修改含中文的文件；用 write 工具或 `[System.IO.File]::WriteAllText(..., UTF8NoBOM)`

## elevated exec 配置修复
- **问题**: `tools.elevated.allowFrom.webchat` 设为 `["agent:main:main"]` 无法匹配运行时发送者身份，导致所有 exec 都需要审批
- **修复**: 改为 `["*"]` 通配，重启网关后生效
- **最终状态**: `tools.elevated.enabled: false`（不再需要 elevated），exec 正常无需审批
