## 📋 1. 项目概览

### 1.1 项目目标和价值

本项目是一个**基于 AI 的量化交易系统**，聚焦于上海期货交易所的黄金期货（AU）30 分钟 K 线波段交易。

**核心价值**：
- 🤖 **AI 驱动决策**：融合技术指标、机器学习预测、LLM 新闻解读三类信号
- 📊 **动态权重调整**：根据市场状态（趋势市/震荡市/高波动市）自适应调整信号权重
- 🔍 **全链路可观测**：每条消息从采集到成交都可追踪（trace_id）
- 🛡️ **完善风控体系**：波动率过滤、连败熔断、最大回撤控制
- 💰 **成本优化**：LLM 缓存机制节省 60% API 成本

### 1.2 核心功能简述

| 功能模块 | 说明 |
|---------|------|
| **数据采集** | CTP 实时行情、新浪财经/东方财富新闻、AKShare 基本面数据 |
| **信号生成** | 技术指标（TA-Lib）+ ML 预测（LightGBM）+ LLM 新闻解读（Claude） |
| **信号融合** | 动态权重融合，至少 2/3 信号一致才交易 |
| **风控管理** | 仓位管理、止损止盈、连败熔断、最大回撤控制 |
| **交易执行** | CTP 接口下单、订单跟踪、成交回报 |
| **监控告警** | Web 实时界面、邮件告警、健康检查 |

### 1.3 技术栈总览

| 类别 | 技术选型 | 说明 |
|-----|---------|------|
| **语言** | Python 3.10+ | 主开发语言 |
| **数据库** | TimescaleDB 2.x | 时序数据存储 |
| **向量库** | Chroma 0.4+ | 新闻 Embedding 存储 |
| **消息队列** | Redis 7.x | 进程间通信 |
| **机器学习** | LightGBM 4.x | 收益率预测 |

| **LLM** | Claude Opus 4 | 新闻解读 |

| **进程管理** | Supervisor | 进程守护 |

### 3.3 关键文件说明

#### ✅ config.py（配置管理）
- **位置**：`quant/common/config.py`
- **功能**：使用 pydantic-settings 管理所有配置项
- **配置项**：数据库、Redis、Chroma、CTP、Claude API、策略参数、日志
- **使用方式**：
  ```python
  from quant.common.config import config
  
  # 访问配置
  db_host = config.database.host
  claude_key = config.claude.api_key
  ```

#### ✅ tracer.py（消息追踪）
- **位置**：`quant/common/tracer.py`
- **功能**：全链路消息追踪，支持 trace_id 生成、事件记录、链路查询
- **核心类**：
  - `MessageTracer`：追踪中间件
  - `TraceContext`：上下文管理器
- **使用方式**：
  ```python
  from quant.common.tracer import MessageTracer, generate_trace_id
  
  tracer = MessageTracer(redis_client, db_client, 'data_collector')
  tracer.trace_event(trace_id, 'kline_received', {'symbol': 'au2406'})
  ```

#### ✅ init_db.py（数据库初始化）
- **位置**：`scripts/init_db.py`
- **功能**：创建 14 张表，启用 TimescaleDB 扩展
- **执行方式**：
  ```bash
  python scripts/init_db.py
  ```

## ⚠️ 6. 风险提示

### 6.1 当前阶段的潜在风险

| 风险 | 影响程度 | 发生概率 | 应对措施 |
|-----|---------|---------|---------|
| **CTP 接口兼容性** | 高 | 中 | 提前测试 vnpy-ctp 版本兼容性 |
| **新闻爬虫反爬** | 中 | 中 | 实现请求频率控制、备用源切换 |
| **数据库性能** | 中 | 低 | TimescaleDB 已优化，设置合理保留策略 |
| **API Key 获取困难** | 高 | 低 | 提前注册，准备备用方案 |

### 6.2 需要注意的事项

1. **CTP 模拟盘限制**：
   - 模拟盘与实盘可能存在延迟差异
   - 某些接口功能在模拟盘不可用
   - 建议尽早申请实盘测试账号（后续阶段）

2. **LLM API 成本**：
   - 建议设置每日调用上限（500 次）
   - 缓存机制第 4 周才会实现，前期成本可能较高
   - 建议监控 API 使用量

3. **数据源稳定性**：
   - 免费数据源可能存在不稳定情况
   - 已设计多源备份机制（AKShare → TuShare）
   - 需持续监控数据质量
